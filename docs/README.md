# docs 目录说明

`docs/` 按"常用资料留根目录、按文档类型分类目录、大型专题独立顶层目录"三层组织。AI 协作规则的唯一入口是仓库根目录 `AGENTS.md`，本目录只保留专项资料和 local-only 过程文档。

## 根目录常用资料

- [项目全景文档.md](项目全景文档.md)：项目全景总览
- [AI使用提示词.md](AI使用提示词.md)：旧 AI 提示词归档入口，当前规则见 `../AGENTS.md`
- [contracts.md](contracts.md)：外部契约与 breaking change 边界
- [ai-docs-migration.md](ai-docs-migration.md)：AI 协作文档迁移记录
- [Bossrush使用物品ID表.md](Bossrush使用物品ID表.md)：常用物品 ID 表
- 飞书应用密钥.md：本地配置（**不要提交远端**）

## 按文档类型分目录

- [设计文档/](设计文档)：玩法、系统、角色、武器等设计稿
- [实现方案/](实现方案)：实现计划、接入方案、规格书、机制说明
- [制作教程/](制作教程)：Unity 制作流程、资源接入教程、操作手册
- [测试分析/](测试分析)：测试结果、性能排查、分析报告
- [版本记录/](版本记录)：历史版本笔记
- [视频策划/](视频策划)：频道日更视频计划、分镜脚本与更新追踪
- [参考资料/](参考资料)：学习笔记和补充资料
- [代码审查/](代码审查)：按日期归档的代码审查报告（原"项目可能的待修复问题/"）

## 按大型专题分目录

仅当一个专题同时拥有"设计 / spec / 实现规划 / 资源"等多阶段文档（体量大）时，才考虑独立顶层目录。这两个目录内部按"阶段"分子目录：

- [末日丧尸模式/](末日丧尸模式)：末日丧尸模式（Zombie Mode）专题
- [未来拓展/](未来拓展)：中长期内容拓展专题

其他专题（龙王/龙皇、幽灵女巫、护士 NPC 等）仍按"文档类型"散落在上述分类目录中。

## AI 工作目录

- `superpowers/{plans,specs}`：由 AI 技能（brainstorming、writing-plans 等）自动写入，不要手动改动其结构。

## 维护约定

- 单一文档优先放入对应类型目录。
- 新增大型专题（拥有 4 阶段文档）才考虑建顶层目录，并按 设计/ spec/ 实现规划/ 资源/ 4 个子目录组织。
- 命名统一：使用「拓展」而非「扩展」。
- 仅经常翻阅的总览/提示词/配置表才放在 `docs/` 根目录。
