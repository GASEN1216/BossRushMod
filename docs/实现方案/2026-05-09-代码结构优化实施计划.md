# 代码结构优化第一阶段 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不改变游戏内行为的前提下，把第一批结构优化落地：固定 runtime module 注册点，收束物品/装备注册入口，并给 Mode D 建立可迁移的模块外壳。

**Architecture:** `ModBehaviour` 继续作为 Unity 生命周期入口，但新增显式 runtime module 注册方法；内容注册先用 `partial ModBehaviour` wrapper 保持私有方法访问和初始化顺序不变；Mode D 先接入无行为 module 外壳，后续再迁移 tick 和清理逻辑。

**Tech Stack:** C# 7.3、Unity Mono、Duckov ModBehaviour、手工 `compile_official.bat` 编译清单、Python 静态 guard。

---

## Scope

本计划只覆盖结构优化第一阶段，目标是建立后续重构可依赖的边界，不迁移具体玩法逻辑，不调整数值，不改变物品、商店、掉落、地图或 UI 文案。

本计划基于方向文档：

- `docs/实现方案/2026-05-09-代码结构优化方向.md`

本阶段完成后，应满足：

- runtime module 在 `OnAwake` 前有显式注册入口。
- 物品配置器注册从 `BossRushIntegration.cs` 抽到专门 wrapper。
- 装备加载和能力系统初始化从 `BossRushIntegration.cs` 抽到专门 wrapper。
- Mode D 有已注册的 runtime module 外壳，但暂不接管 Mode D 行为。
- guard 覆盖新增结构，避免后续绕回旧写法。

## File Structure

### Create

- `Common/Lifecycle/BossRushRuntimeModuleBase.cs`
  为 runtime module 提供 no-op 默认生命周期实现，避免每个 module 重复空方法。

- `Common/Lifecycle/ArchitectureSentinelRuntimeModule.cs`
  无行为哨兵模块，用来证明 runtime module 注册链路稳定存在。

- `Common/Lifecycle/BossRushRuntimeModuleRegistration.cs`
  `ModBehaviour` partial 文件，集中注册第一阶段 runtime module。

- `Integration/Items/ItemContentRegistry.cs`
  `ModBehaviour` partial 文件，集中维护所有 `ItemFactory.RegisterConfigurator` 调用，保持当前顺序。

- `Integration/EquipmentContentRegistry.cs`
  `ModBehaviour` partial 文件，集中维护装备加载、武器预热、装备能力系统初始化和清理，保持当前顺序。

- `ModeD/ModeDRuntimeModule.cs`
  Mode D runtime module 外壳。第一阶段只保存 owner 引用，不执行行为。

- `tests/ContentRegistryGuard.py`
  静态验证内容注册已从 `BossRushIntegration.cs` 收束到 registry wrapper。

### Modify

- `ModBehaviour.cs`
  在 `runtimeModuleHost.OnAwake(this);` 前调用 `RegisterRuntimeModules();`。

- `Integration/BossRushIntegration.cs`
  用 wrapper 调用替换物品配置器注册块、装备加载块、装备能力初始化块、装备能力清理块。

- `compile_official.bat`
  添加新增 `.cs` 文件到官方编译清单。

- `tests/ArchitectureStructureGuard.py`
  增加新增 lifecycle 文件、注册调用和注册顺序验证。

"""ContentRegistryGuard: item and equipment registration must be routed through registry wrappers."""

from pathlib import Path
import re
import sys


COMPILE = Path("compile_official.bat")
INTEGRATION = Path("Integration/BossRushIntegration.cs")
ITEM_REGISTRY = Path("Integration/Items/ItemContentRegistry.cs")
EQUIPMENT_REGISTRY = Path("Integration/EquipmentContentRegistry.cs")


def fail(message: str) -> int:
    print(message)
    return 1


def normalize(text: str) -> str:
    return re.sub(r"/+", "/", text.replace("\\", "/"))


def main() -> int:
    compile_text = normalize(COMPILE.read_text(encoding="utf-8", errors="ignore"))
    integration_text = INTEGRATION.read_text(encoding="utf-8", errors="ignore")

    required_sources = [
        "Integration/Items/ItemContentRegistry.cs",
        "Integration/EquipmentContentRegistry.cs",
    ]
    for source in required_sources:
        if source not in compile_text:
            return fail("ContentRegistryGuard: compile_official.bat missing source: " + source)

    if not ITEM_REGISTRY.exists():
        return fail("ContentRegistryGuard: missing " + str(ITEM_REGISTRY))

    item_registry_text = ITEM_REGISTRY.read_text(encoding="utf-8", errors="ignore")
    for required in [

        "BrickStoneConfig.RegisterConfigurator();",

        "PeaceCharmConfig.RegisterConfigurator();",

        "AchievementMedalConfig.RegisterConfigurator();",

        "AwenLootSweepTokenConfig.RegisterConfigurator();",

        "RespawnItemConfig.RegisterConfigurators();",
        "BloodhuntTransponderConfig.RegisterConfigurator();",
        "FoldableCoverPackConfig.RegisterConfigurator();",
        "ReinforcedRoadblockPackConfig.RegisterConfigurator();",

        "EmergencyRepairSprayConfig.RegisterConfigurator();",
        "ZombieTideInvitationConfig.RegisterConfigurator();",
        "ZombieTideBeaconConfig.RegisterConfigurator();",
        "ItemFactory.RegisterConfigurator(ADVENTURE_JOURNAL_TYPE_ID, OnAdventureJournalLoaded);",

    return 0


if __name__ == "__main__":
