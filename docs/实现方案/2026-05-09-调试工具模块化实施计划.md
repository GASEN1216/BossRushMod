# 调试工具 Runtime Module Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不改变游戏内行为的前提下，把方向文档“第一批建议任务 B”的横切热键和调试 UI 外壳接入 runtime module。

**Architecture:** `ModBehaviour` 继续持有所有既有调试 UI 字段和私有方法；新增一个 `DebugToolsRuntimeModule` 只负责接收 `OnUpdate` / `OnLateUpdate` 并调用 `ModBehaviour` wrapper。wrapper 保持原有调用顺序，不迁移 UI 实现、字段、数值或热键定义。

**Tech Stack:** C# 7.3、Unity Mono、Duckov ModBehaviour、手工 `compile_official.bat` 编译清单、Python 静态 guard。

---

## Scope

本计划只做“外壳模块化”：

- 移出 `ModBehaviour.Update` 中的调试热键 / 调试 tick 直接调用。
- 移出 `ModBehaviour.LateUpdate` 中的调试 UI late update 直接调用。
- 保留 `OnGUI` 绘制入口不动，避免影响 IMGUI 绘制时机。
- 保留 F2/F3/F4/F5/F6/F7/F8/F9/F10/F12 等既有热键行为和 DevMode 判断不动。

不做：

- 不重写 F3 调试菜单。
- 不改 BossPool / NPC Teleport / ItemSpawner 的 UI 行为。
- 不改 `HardcodedDevModeEnabled`。
- 不迁移成就热键；它是玩家可配置入口，不属于 DevMode 调试外壳。

- `tests/ArchitectureStructureGuard.py`
  增加新增文件、注册、wrapper 转发和主入口不再直接调用调试工具的验证。

## Task 2: Runtime Module And Wrappers

- [ ] **Step 1: Create runtime module**

Create `DebugAndTools/DebugToolsRuntimeModule.cs`:

```csharp
namespace BossRush
{
    internal sealed class DebugToolsRuntimeModule : BossRushRuntimeModuleBase
    {
        private ModBehaviour owner;

        public override string ModuleName
        {
            get { return "DebugTools"; }
        }

        public override void OnAwake(ModBehaviour owner)
        {
            this.owner = owner;
        }

        public override void OnUpdate(float deltaTime, float unscaledDeltaTime)
        {
            if (owner == null)
            {
                return;
            }

            owner.TickDebugTools(deltaTime, unscaledDeltaTime);
        }

        public override void OnLateUpdate()
        {
            if (owner == null)
            {
                return;
            }

            owner.LateUpdateDebugTools();
        }

        public override void OnDestroy()
        {
            owner = null;
        }
    }
}
```

- [ ] **Step 2: Create wrapper file**

Create `DebugAndTools/DebugToolsRuntimeHooks.cs`:

```csharp
namespace BossRush
{
    public partial class ModBehaviour
    {
        internal void TickDebugTools(float deltaTime, float unscaledDeltaTime)
        {
            UpdateFpsCounter();
            UpdateMapClickDebug();
            CheckBossPoolWindowHotkey();
            CheckItemSpawnerHotkey();
            CheckF3DebugCheatMenuHotkey();
            TickF3DebugCheatMenu();
        }

        internal void LateUpdateDebugTools()
        {
            BossPoolLateUpdate();
            NPCTeleportUILateUpdate();
            F3DebugCheatMenuLateUpdate();
        }
    }
}
```

The parameters are intentionally accepted for parity with runtime module lifecycle even though existing debug methods use Unity `Time` directly.

- [ ] **Step 3: Register module**

Modify `Common/Lifecycle/BossRushRuntimeModuleRegistration.cs`:

private void RegisterRuntimeModules()
