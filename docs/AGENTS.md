# docs/AGENTS.md — 文档目录专项规则

> 先读根目录 `AGENTS.md`。`docs/` 默认 local-only，是否纳入 git 由 owner 决定。

## 规则

- 不把 `docs/` 中的历史计划、视频稿、旧设计稿当作当前代码事实；它们只能作为线索。
- 涉及架构约定的长期有效规则放 `docs/架构说明/`。
- 迁移、契约、协作说明放 `docs/ai-docs-migration.md`、`docs/contracts.md` 等明确命名文件。
- 敏感资料如 `docs/飞书应用密钥.md` 不要复制到回答、提交或公开文档。
- 新文档如果包含未确认事实，标 `Needs owner confirmation`。
- 文档引用路径要使用仓库内真实路径；旧路径转发时说明新 canonical 文件。
